使用方式

在项目less目录下的 baseUI “桥接模块”中，选择性引入项目需要的基础 UI 文件即可。

        // Element
        // ---------------------------------------
        @import "../lib/baseUI/element/button";
        ...

        // Component
        // ---------------------------------------
        @import "../lib/baseUI/component/dialog";
        ...

组成部分

baseUI 主要由以下几个部分组成：命名约定（naming）、基础UI（element、component）。

naming

常用 Web 界面构建中类名命名约定

原则：不混合使用分隔符，不使用无语义的名称。抽象类型按“功能”去命名，抽象形态按“表现形式”去命名。

CSS类名中的分隔符选择：

  • ① -（中横杆） √
  • ② _（下划线）
  • ③ redBtn（驼峰命名法）
  • ④ block__element--modifier（BEM命名法）

CSS类名中的语义化“关键字”及使用方式：

        // 使用拼接方式组成所需要的类名

        .btn {} // 按钮
        .btn-default {} // 默认主题按钮
        .btn-green {}   // 绿色按钮
        .btn-lg {}   // 大的按钮
        .btn-group {}   // 按钮组
